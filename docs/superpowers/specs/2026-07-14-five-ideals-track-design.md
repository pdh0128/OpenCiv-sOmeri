# 5대이념 트랙 (서브프로젝트 4) 설계

**Date:** 2026-07-14
**Status:** Approved (design), pending implementation plan

## Background

서브프로젝트 3(삼원체제 정부) 완료·머지됨. 다음은 원래 계획의 서브프로젝트 4 — 시조왕의 유훈 "통합·지식·발전·질서·개척"(五志) 다섯 이념을 실제 게임 트랙으로 만든다.

## 결정된 방향

- **다섯 이념 각각 누적 포인트로 추적.** Civ5의 "역사 포인트"류 트랙과 비슷하게, 플레이어 행동이 특정 이념에 포인트를 쌓는다. 이번 범위에서는 **이미 존재하는 게임 행동에서 자동으로 포인트가 쌓이는 방식**만 구현한다(플레이어가 직접 "이념 선택"을 하는 게 아니라, 이미 하고 있는 플레이 스타일이 자연히 다섯 이념 중 어디에 쌓이는지 반영) — 이래야 서브프로젝트 5(황금/암흑시대)가 "어느 이념이 우세한가"를 판단 기준으로 쓸 수 있다.
- **이념 ↔ 실제 게임 행동 매핑** (실록의 다섯 이념 원문 근거 그대로):
  - **통합(Unity)**: 도시 수(정착 행동). 도시를 정착할 때마다 획득.
  - **지식(Knowledge)**: 연구 완료(서브프로젝트 2c). 기술 하나 연구 완료할 때마다 획득.
  - **발전(Development)**: 건물 완공(서브프로젝트 2a). 건물 하나 완공할 때마다 획득.
  - **질서(Order)**: 정부 분과 선택 유지 턴 수(서브프로젝트 3). 같은 분과를 유지한 채 턴이 지날 때마다 소량씩 획득 — "질서"는 안정을 뜻하므로 자주 바꾸면 안 쌓인다.
  - **개척(Pioneering)**: 유닛의 미개척 타일 최초 방문. 유닛이 이전에 아무도 밟지 않은 타일에 처음 도달할 때마다 획득.
- **점수는 플레이어 단위로 누적**, 절대 감소하지 않는다(단, 질서는 분과를 바꾸면 그 턴의 획득이 스킵될 뿐 기존 누적치가 깎이진 않음).
- **UI**: 기존 `ResearchDisplayInfo`/`GovernmentDisplayInfo`와 같은 패턴으로, 새 상태바 항목이나 별도 패널에 다섯 이념 현재 점수를 표시만 한다. 이번엔 이걸로 뭘 "선택"하게 하지 않는다 — 순수 집계+표시. (서브프로젝트 5가 이 점수를 읽어서 황금/암흑시대를 판정한다.)

## 데이터 모델

새 상수 5개 정의(코드 내, 별도 config 불필요 — 실록 유래 고정 이름이라 데이터 파일화할 이유 없음): `unity`, `knowledge`, `development`, `order`, `pioneering`.

`Player`에 신규 필드 `private idealPoints: Record<string, number>` — 5개 키를 0으로 초기화.

## 트리거 지점 (기존 코드에 훅 추가)

| 이념 | 트리거 위치 | 획득량 |
|---|---|---|
| 통합 | `UnitActions.settleCity()`의 `onAction` — 도시 생성 직후 | +10 |
| 지식 | `Player.processResearchTurn()` — 기술 완료 분기 안 | +15 |
| 발전 | `City.processProductionTurn()` — 건물 완료 분기 안 | +10 |
| 질서 | `Player`의 `nextTurn` 리스너 — `selectedGovernmentBranch`가 있고 직전 턴과 동일하면 | +2 |
| 개척 | `Tile`에 "방문 여부" 플래그 추가, 유닛이 새 타일에 진입할 때(`Unit.moveToTile()`) 미방문 타일이면 | +5 |

`Player`가 각 트리거 지점에 직접 접근하기 애매한 경우(예: City가 Player의 이념 점수를 올려야 함), 기존에 이미 있는 `this.player`(City→Player) / 콜백 참조를 그대로 활용 — 새 통신 경로 신설 없음.

## 통신 프로토콜

`Player.toJSON()`에 `idealPoints` 필드를 추가해, 기존 `updateProductionQueue`/`updateResearchQueue`류와 마찬가지로 값이 바뀔 때마다 `updateIdealPoints` 브로드캐스트(`{ idealPoints: Record<string, number> }`).

## UI

`StatusBar`에 다섯 이념 점수 요약을 여는 버튼 하나 추가(예: 기존 라벨 옆 "이념" 텍스트 버튼) → 새 `IdealsDisplayInfo` 패널, 다섯 항목 정적 리스트로 현재 점수 표시. 상호작용(클릭해서 뭔가 선택) 없음 — 순수 조회 패널.

## 범위 밖

- 이념 점수를 실제로 소비하는 효과(승리 조건, 정책 해금 등) — 서브프로젝트 5(황금/암흑시대)에서 다룸.
- 개척 이념의 "미방문 타일" 플래그를 서버가 영속화하는 저장/불러오기 — 이번 세션에서 저장/로드 시스템 자체가 없으므로 범위 밖(게임 한 판 동안만 유효).

## 리스크 / 열린 사항

- 점수 획득량(+10/+15/+2/+5 등)은 대략값 — 서브프로젝트 5가 실제 임계값을 정할 때 함께 조정.
- "개척" 트리거(미방문 타일 진입)는 `Tile` 클래스에 새 상태(`visited: boolean`)를 추가해야 함 — 기존 파일 수정이지만 필드 하나 추가하는 최소 변경.
